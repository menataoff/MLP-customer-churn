# MLP для прогнозирования оттока клиентов

## Описание задачи
Бинарная классификация: уйдёт клиент или нет (дисбаланс 20/80)

## Данные
- Bank Customer Churn Dataset (10 000 записей)
- Признаки: кредитный рейтинг, возраст, баланс, зарплата, география, пол и др.

## Методы борьбы с дисбалансом
- SMOTE
- Стратифицированное разбиение
- pos_weight в функции потерь

## Архитектура MLP
- 32 → 16 → 1
- BatchNorm + Dropout (0.5)

## Результаты
| Модель | Accuracy | Precision | Recall | F1 | ROC-AUC | PR-AUC |
|--------|----------|-----------|--------|-----|---------|--------|
| MLP | 0.813 | 0.715 | 0.530 | 0.61 | 0.862 | 0.708 |
| CatBoost | 0.860 | 0.681 | 0.582 | 0.63 | 0.870 | 0.723 |
| Random Forest | 0.844 | 0.616 | 0.612 | 0.61 | 0.857 | 0.673 |

## Выводы
- MLP показал результат, сопоставимый с градиентным бустингом
- PR-AUC > 0.7 подтверждает эффективность модели при дисбалансе
- Рекомендуемый порог: 0.5 (баланс precision/recall)
